## 数学原理

### 核心模型：平面单应性（Plane Homography）

你看到的”近大远小”格子，在数学上被称为**平面单应性投影**。整个过程分解为三步：

### 第一步：物理世界网格

假设现实世界的地面是一个**绝对平坦的二维笛卡尔坐标系**，以 $2\text{m} \times 2\text{m}$ 为单位建立均匀网格：

$$W_i = \begin{bmatrix} X_i \\ Y_i \end{bmatrix}$$

其中 $X_i, Y_i \in \{\dots, -4, -2, 0, 2, 4, \dots\}$。此时这些点是标准的正方形格网，没有任何透视特征。

### 第二步：中心化与齐次坐标

UTM 坐标数值巨大（$812820 \sim 812853$），直接计算会导致浮点精度下溢。脚本对世界坐标做**中心化**和**齐次化**：

1. **中心化减去均值：**

$$W_{ci} = W_i - \text{world\_mean}$$

2. **升维至齐次坐标：**

$$\mathbf{x}_{\text{world}} = \begin{bmatrix} X_{ci} \\ Y_{ci} \\ 1 \end{bmatrix}$$

### 第三步：单应性变换（核心）

$H$ 是一个 $3 \times 3$ 的**非奇异线性变换矩阵**，代表”像素平面 → 地平面”的映射。

**正向（像素 → 世界）：**

$$\begin{bmatrix} x' \\ y' \\ w' \end{bmatrix} = H \cdot \begin{bmatrix} u \\ v \\ 1 \end{bmatrix}, \quad
X_{\text{world}} = \frac{x'}{w'} + \text{world\_mean}_x, \quad
Y_{\text{world}} = \frac{y'}{w'} + \text{world\_mean}_y$$

**反向（世界 → 像素，用于网格绘制）：**

$$\begin{bmatrix} u' \\ v' \\ w' \end{bmatrix} = H^{-1} \cdot \begin{bmatrix} X_{ci} \\ Y_{ci} \\ 1 \end{bmatrix}, \quad
u = \frac{u'}{w'}, \quad v = \frac{v'}{w'}$$

其中最后一步除以 $w'$ 称为**透视除法（Perspective Division）**——当 $w'$ 随空间深度非线性变化时，原本等间距的平行线在像素画面中自动产生向心收缩。

### 消失点（Vanishing Point）

在射影几何中，两条平行的直线在三维欧氏空间里永远不会相交。但通过 $H^{-1}$ 映射到二维成像面后，其齐次分母 $w' \to 0$，使得所有平行线在像素画面中**必然交汇于一点**，即消失点。

> 格子收缩得越快 → 相机俯仰角越小（越平）；格子收缩得越慢 → 相机越接近垂直向下俯视。

## 精度评估

地面基本平坦（工地硬化路面），模型假设合理。

### 整体指标

| 指标 | 数值 |
|------|------|
| 原始控制点 | 26 对 |
| RANSAC 剔除 | 3 对（阈值 1.5 m）|
| 最终控制点 | **23 对** |
| **RMSE（世界坐标）** | **34.3 cm** |
| MAE（世界坐标） | 25.4 cm |
| 最大重投影误差 | 78.0 cm |
| RMSE（像素空间） | 17.7 px |
| RANSAC 剔除带来的精度提升 | **87%**（无 RANSAC 时 RMSE = 2.66 m）|

### 按图像区域分组

| 区域 | 点数 | RMSE | 误差范围 |
|------|------|------|----------|
| 远端 / 画面上方（远处地面）| 4 | **61.7 cm** | 42 ~ 78 cm |
| 中远 / 画面中上 | 7 | 37.4 cm | 4 ~ 68 cm |
| 中近 / 画面中下 | 6 | **9.6 cm** | 6 ~ 16 cm |
| 近端 / 画面下方（近处地面）| 6 | 15.7 cm | 5 ~ 30 cm |

### 按世界距离分组（距相机约）

| 距离 | 点数 | RMSE |
|------|------|------|
| < 10 m | 7 | **13.2 cm** |
| 10 ~ 15 m | 9 | **14.2 cm** |
| 15 ~ 20 m | 7 | **58.5 cm** |

### 误差来源分析

1. **透视放大效应**（主因）：画面远端 1 像素 ≈ 5~10 cm 世界距离，而近端 1 像素 ≈ 1~3 cm。同样的像素标注偏差，在远端被放大 3~5 倍
2. **控制点分布不均**：23 个点集中在画面中部，远端只有 4 个点且标注精度偏低
3. **地面微起伏**：工地虽基本平坦，但局部有 ±5~10 cm 的不平整
4. **RTK 测量精度**：RTK 本身有 ~2 cm 的定位误差

### 结论

| 距离范围 | 定位精度 | 适用场景 |
|----------|----------|----------|
| 0 ~ 15 m | **~15 cm** ✅ 良好 | 人员/车辆精确定位、距离报警 |
| 15 ~ 20 m | **~60 cm** ⚠️ 一般 | 区域粗略判断、存在性检测 |
| > 20 m | 不建议依赖 | 超出控制点覆盖范围 |

> 整体 RMSE = **34 cm**，对于工地安全监控（人员闯入检测、机械距离预警）是够用的。
> 如果未来需要 < 20 cm 的全场景精度，建议：① 在远端补标 3~5 个控制点；② 考虑使用完整的相机内参+外参模型（PnP）替代单应性。

## 使用方法

```bash
# 第一步：加载原始数据，RANSAC 清洗离群点，视频流验证
uv run python calibration/compute_calib.py

# 第二步：用清洗后数据计算最终 H 矩阵，可视化透视网格，保存 .npz
uv run python calibration/compute_and_grid_vis.py
```

标定结果保存在 `calibration/camera_calib_371.npz`，可在推理脚本中直接加载：

```python
import numpy as np
calib = np.load(“calibration/camera_calib_371.npz”)
H = calib[“H”]             # 3x3 单应性矩阵 (像素 → 中心化世界)
world_mean = calib[“world_mean”]  # 世界坐标均值 (2,)
```

### 推理时使用 H 矩阵

```python
# 像素坐标 → UTM 世界坐标
def pixel_to_world(u, v, H, world_mean):
    p = np.array([u, v, 1.0])
    wp = H @ p
    x, y = wp[0] / wp[2], wp[1] / wp[2]
    return x + world_mean[0], y + world_mean[1]

# UTM 世界坐标 → 像素坐标（用于在画面上绘制）
def world_to_pixel(wx, wy, H_inv, world_mean):
    wc = np.array([wx - world_mean[0], wy - world_mean[1], 1.0])
    pp = H_inv @ wc
    return pp[0] / pp[2], pp[1] / pp[2]
```
